## Tareas fuera de alcance (v1.1)

Las siguientes tareas NO están en este `tasks.md` pero aterrizarán en un cambio de seguimiento. Cada una se nombra con el slug de cambio que el orquestador usará cuando el usuario abra el trabajo de v1.1.

- **Attachments** (`transactions-attachments`): modelo Prisma `Attachment` + port `AttachmentStorage` + adaptador `LocalDiskAttachmentStorage` + env var `ATTACHMENTS_DIR` + tres rutas Hono (`GET/POST/DELETE /api/transactions/:id/attachments`). La interface del port desde design §DG-TX-5 es el contrato vinculante.
- **Recurrence** (`transactions-recurrence`): modelo Prisma `RecurrenceRule` con `frequency | interval | byMonthDay | byDay` + generación on-demand al cargar el dashboard + `idempotencyKey` en cada fila generada.
- **Transfers** (`transactions-transfers`): aggregate `Transfer` O link `transferGroupId` + semántica de escritura atómica de dos filas.
- **Idempotency keys** (`transactions-idempotency`): campo `idempotencyKey` en `Transaction` + `@@unique([userId, idempotencyKey])` + middleware que lee el header `Idempotency-Key`.
- **Bulk import / CSV upload** (`transactions-bulk-import`): `POST /api/transactions/bulk` con rate limit + idempotency key por fila + el campo `idempotencyKey` del slice previo.
- **Archivo histórico de FX para transacciones back-dated** (`fx-archive`): un archivo de snapshots de DolarAPI (Upstash o Postgres) que permita a filas con `transactionDate` en el pasado usar la rate en ese momento, no la rate al momento de la escritura.
- **UI de calidad de producción** (`transactions-ui`): la smoke UI bajo `app/transactions/` se reemplaza con una UI de producción; filtros, charts, export-to-CSV.
- **Reglas de captura de Sentry para los nuevos eventos del logger** (baja prioridad; si un futuro cambio `sentry-tuning` surface una necesidad, los eventos `transactions.create`, `transactions.update`, `transactions.delete`, `transactions.fx.convert` ganan reglas de captura ahí).
