## Out-of-scope tasks (v1.1)

The following tasks are NOT in this `tasks.md` but will land in a follow-up change. Each is named with the change slug the orchestrator will use when the user opens the v1.1 work.

- **Attachments** (`transactions-attachments`): `Attachment` Prisma model + `AttachmentStorage` port + `LocalDiskAttachmentStorage` adapter + `ATTACHMENTS_DIR` env var + three Hono routes (`GET/POST/DELETE /api/transactions/:id/attachments`). The port interface from design §DG-TX-5 is the binding contract.
- **Recurrence** (`transactions-recurrence`): `RecurrenceRule` Prisma model with `frequency | interval | byMonthDay | byDay` + on-demand generation at dashboard load + `idempotencyKey` on each generated row.
- **Transfers** (`transactions-transfers`): `Transfer` aggregate OR `transferGroupId` link + atomic two-row write semantics.
- **Idempotency keys** (`transactions-idempotency`): `idempotencyKey` field on `Transaction` + `@@unique([userId, idempotencyKey])` + middleware that reads `Idempotency-Key` request header.
- **Bulk import / CSV upload** (`transactions-bulk-import`): `POST /api/transactions/bulk` with rate limit + idempotency key per row + the `idempotencyKey` field from the previous slice.
- **Historical FX archive for back-dated transactions** (`fx-archive`): a DolarAPI snapshot archive (Upstash or Postgres) that allows back-dated `transactionDate` rows to use the rate at the time, not the rate at the moment of write.
- **Production-quality UI** (`transactions-ui`): the smoke UI under `app/transactions/` is replaced with a production UI; filters, charts, export-to-CSV.
- **Sentry capture rules for the new logger events** (low-priority; if a future `sentry-tuning` change surfaces a need, the `transactions.create`, `transactions.update`, `transactions.delete`, `transactions.fx.convert` events gain capture rules there).
