## Goal

`sdd-apply` for `accounts-ledger` lands the full `accounts` capability (canonical at `openspec/specs/accounts/spec.md` after sync) in three chained PRs against `develop`:

- **PR-A** lands the Prisma model + 5 enums + migration + the domain layer (enums, value object, service skeleton) + 2 new error codes. The `accounts` module directory exists; the data model is the only durable surface the next two PRs build on.
- **PR-B** lands the application layer (7 actions + 4 Zod schemas) + the infrastructure layer (Prisma repository adapter + `FxRateProvider` stub) + the Hono routes (7 endpoints + `requireSession` middleware + 2 more error codes) + integration tests. The API is end-to-end functional at this point and `pnpm test` is green on the API surface.
- **PR-C** lands the smoke UI (3 Server Components + 2 Client Components + typed Hono client + Tailwind v4 setup) + the Spanish mirror. The hand-verifiable acceptance criteria from the proposal (#4–#7) close in this PR.

After the three PRs merge to `develop`, the canonical spec lands at `openspec/specs/accounts/spec.md` via `sdd-sync`, and `accounts-ledger` is archived.
