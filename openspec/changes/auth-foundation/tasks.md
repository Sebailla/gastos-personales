## Goal

`auth-foundation` lands a complete, production-grade identity layer
for `gastos-personales`. When `sdd-apply` finishes, the system must
expose seven HTTP endpoints (`POST /auth/register`, `POST /auth/login`,
`POST /auth/refresh`, `POST /auth/logout`, `GET /auth/me`,
`GET /auth/oauth/google`, `GET /auth/oauth/google/callback`) backed by
three SQLite tables (`users`, `refresh_tokens`, `oauth_accounts`),
with Argon2id password hashing, HS256 access JWTs, opaque rotated
refresh tokens with family-cascade revocation, Google OAuth 2.0
with email-match auto-link, an `authMiddleware` that pins
`alg: HS256` and rejects `alg: none`, and ≥ 80 % line + branch
coverage on the `auth` module — all gated by `bun test`, `lint`,
`typecheck`, and `gga run`.

## Scope summary

- 3 Drizzle tables + versioned migration file (Drizzle Kit generated).

### Phase 7 — Security tests (dedicated suite, adversarial review input)

- [ ] **T-032** Security test suite (timing, reuse, state CSRF, alg confusion, secrets in logs)
  - **Scope**: five focused integration tests in
    `src/modules/auth/__tests__/security/`:
    1. `login.timing.test.ts`: with a real Argon2id hash
       and a fixed dummy hash, the action's response time
       for the "wrong password" branch and the "unknown
       email" branch are within a documented statistical
       threshold (e.g. Welch's t-test p > 0.01 over 30
       samples). The test runs in CI; in local dev a
       `--skip-timing` flag tolerates noisy machines.
    2. `refresh.reuse.test.ts`: with a real in-memory
       SQLite, `rotate(t)` succeeds, then `rotate(t)`
       again returns `REFRESH_REVOKED` and the family is
       revoked atomically (a third token in the same
       family also fails). Two concurrent `rotate(t)`
       calls both end in failure for the loser (the
       design's "two legitimate clients with the same
       refresh is itself a theft signal" rule).
    3. `oauth.state-csrf.test.ts`: a callback with a
       missing `oauth_state` cookie redirects to
       `oauth_state_mismatch`; with a tampered state
       redirects to `oauth_state_mismatch`; with an
       expired cookie (Max-Age 10 min) redirects to
       `oauth_state_mismatch`. No `users` row is created
       and no `oauth_accounts` row is inserted in any
       failure case (asserted by a row count).
    4. `jwt.algorithm-confusion.test.ts`: a token with
       `alg: none` (hand-rolled, valid base64url
       header+payload, empty signature) is rejected by
       `authMiddleware`. A token with `alg: RS256` and
       the HS256 secret as the "RSA public key" is
       rejected.
    5. `secrets.in-logs.test.ts`: a request that
       includes a `password`, a `refresh_token`, an
       `Authorization: Bearer <jwt>` header, or a `code`
       query parameter does not cause any of those values
       to appear in the captured log output across the
       register, login, refresh, logout, and OAuth
       callback paths.

## Risks specific to apply

| Risk | Lives in | Mitigation |

| `@node-rs/argon2` fails to load on Bun. | T-011 | The benchmark script in T-011 is the smoke test. If it crashes, fall back to `node-argon2` (acceptable per the design's alternatives) and record the choice in the apply-progress. |
| `jose` ESM import shape breaks the Bun loader. | T-012 | The test in T-012 imports `jwtVerify` directly; if Bun's loader chokes, the test fails before any production code is written. Fall back to `fast-jwt` (also in the design's alternatives). |
| `arctic` v2.x ships breaking changes between minor versions. | T-013, T-021 | Pin `arctic` to a minor version in `package.json`; the test in T-021 stubs the network calls so the library surface used in production is small. |

## Out-of-scope for this change

The following are tracked in the proposal and design as separate
changes. The `sdd-apply` worker must **not** smuggle any of these
into this slice.

- **Other OAuth providers** (Apple, Facebook, GitHub). Tracked as a
  post-MVP change. The data model supports them; this change
  ships with `provider = 'google'` only.
- **Password reset and email verification**. Tracked as
  `auth-password-reset`. The refresh-on-password-change revoke
  behavior is part of that change, not this one. The
  `RefreshTokenRepository.revokeAllForUser(userId)` helper is
  not built in this change; if a future task requires it, it
  belongs to `auth-password-reset`.
- **Multi-factor authentication**. Post-MVP.
- **Rate limiting on `/auth/login`, `/auth/register`,
  `/auth/oauth/google`**. Tracked as `security-rate-limiting`.
  The login action's user-enumeration mitigation (BR-AUTH-07) is
  *not* rate limiting and *is* in scope; see T-023 and T-032.
- **Session listing and "log out all devices"**. The logout
  endpoint in this change revokes only the current refresh chain
  (BR-AUTH-04). Listing all chains and bulk-revoking is a
  separate change.
- **Generic RBAC on top of `user_id`**. Every later change
  handles its own `WHERE user_id = ?` discipline.
- **UI screens** (login form, register form, OAuth button).
  Tracked as `ui-auth-shell`. The contract in this change is the
  HTTP API only.
- **On-demand account linking** ("Link Google to my account"
  from settings). The auto-link flow at first OAuth login is
  the only linking path; the profile-driven flow is a separate
  change.
- **"Unlink Google" / "Set password" actions**. Separate change.
- **Refresh token pruning**. Revoked tokens accumulate in the
  DB; a future change prunes them. Not in scope here.
- **User deletion**. Tracked as `user-deletion` (GDPR-style
  cleanup). Not in scope here.
- **Change-email flow**. `users.email` is immutable in MVP.
- **Email notifications on auto-link**. Future hardening pass.
- **Deployment to Fly.io**. Tracked as `fly-deploy`. The CI
  workflow in T-033 runs on every push; deploy is a separate
  pipeline. The Argon2 benchmark in T-011 is re-run on the
  target VM in `fly-deploy` to confirm the 50–100 ms hash time.
- **Welcome email on `UserRegistered`**. The event is dispatched
  (T-016, T-026); a downstream worker in a later change consumes
  it. No consumer ships in this change.
